ZPT-JS reference - Attributes - I18nLanguage

  • Syntax.
  • Description.
  • Differences with ZPT.
  • Examples.

Syntax

argument ::= expression

Description

The data-ilanguage statement defines an expression that evaluates to the current language to use in translations. After using a data-ilanguage statement you will be able to use i18nBundle instances in data-idomain attributes.

ZPT-JS uses the tr expression to make it easy to translate text, number, dates...

Differences with ZPT

This statement does not exist in ZPT.

Examples

Using a literal: 

<div data-ilanguage="'es'">
    ...
</div>

Using a path expression: 

<div data-ilanguage="myObject/getLanguage()">
    ...
</div>
