Syntax

argument       ::= props_item [';' props_item]*
props_item     ::= ('required') variable_name (variable_type (expression))
variable_name  ::= Name
variable_type  ::= Name

Description

The data-props statement declares a list of variables and some settings about them.

Each declaration can define:

  • The required reserved word declares a required variable. It is optional. The default value is false.
  • The variable name. It is the only mandatory item.
  • The variable type (optional). It declares the type of the variable.
  • An expression working as default value (optional). It is evaluated only if the value of the variable is undefined.

Some example of types:

Type Value example
undefined undefined
null null
number NaN
number 5
array []
string ''
function function(){}
regexp /a/
date new Date()
object {}
myconstructor new MyConstructor()

A strict mode is set to true inside the node with this attribute.

Differences with ZPT

This statement does not exist in ZPT.

Examples

Declaring a number, a string, an array and a date: 

data-props="aNumber number;
            aText string;
            AnArray array;
            ADateValue date"

Declaring a number, a string, an array and a date (all required): 

data-props="required aNumber number;
            required aText string;
            required AnArray array;
            required ADateValue date"

Declaring some default values: 

data-props="aNumber number 99;
            aText string 'a default value'"
